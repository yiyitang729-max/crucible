# Crucible V1 MVP 效果测评方案

> 测评目标：验证 V1 四个 Plan 的端到端功能，发现需要调优的问题，为作品集准备用户故事素材。
> 预计耗时：30-40 分钟

---

## 一、测评前准备

1. 确认飞书 Bot 在线（发任意消息能收到回复）
2. 确认本地环境就绪：
   ```bash

   ```

## 二、测评用例

### 用例 1：纯想法 → 完整追问 → 笔记生成

**目的：** 验证核心路径（Router → Socratic 3 轮追问 → 收尾出笔记）

**发送：**
> 我觉得很多人学东西只停留在「收藏」，从来不回看

**预期行为：**
- [ ] Bot 回复第 1 个追问（锚定），语气自然，不列选项
- [ ] 回复后 Bot 追问第 2 个问题（深挖），往你的具体经历引导
- [ ] 继续回复，Bot 第 3 轮自然收尾或直接帮你总结
- [ ] 最多 3 轮追问（不超过 3 个问题）
- [ ] 收尾消息包含「✅」+ 笔记格式（日期/洞察/原始触发/追问摘要/个人应用场景）
- [ ] 追问语气像朋友聊天，不像问卷或老师

**你的回复参考（不用完全照搬，自然聊就行）：**
- 第 1 轮回复：「就是我自己，微信收藏了一堆文章，从来没打开过」
- 第 2 轮回复：「比如上次看到一篇关于 AI PM 能力模型的文章，收藏了但一直没看，后来面试被问到完全没准备」

**记录：**
- 实际追问了几轮？
- 哪句追问最自然 / 最生硬？
- 收尾笔记内容是否准确？

---

### 用例 2：带 URL → 追问 → 笔记生成

**目的：** 验证 URL 输入的 Router 判断 + 锚定问题差异

**发送：** 随便找一篇你最近看过的文章链接，附带一句你的看法。比如：
> https://xxxxxx 这篇讲 AI 产品经理的，我觉得说得挺对的

**预期行为：**
- [ ] Bot 识别为 content_digest
- [ ] 因为你同时给了 URL + 看法，Bot 应该跳过锚定，直接深挖
- [ ] 追问围绕你的看法展开，不是问「你最想搞清楚哪一点」
- [ ] 最终收尾出笔记

**记录：**
- Bot 是否正确跳过了锚定？
- 追问是否围绕你的看法而非文章内容？

---

### 用例 3：主动结束对话

**目的：** 验证 conversation_end 路由 + 提前收尾笔记

**发送：**
> 最近觉得写作其实是一种思考工具

**Bot 追问后，直接说：**
> 先到这里吧

**预期行为：**
- [ ] Bot 识别「先到这里」为 conversation_end
- [ ] 回复包含总结 + 笔记格式
- [ ] 不会继续追问

**记录：**
- 收尾是否干脆？有没有继续追问？

---

### 用例 4：提前收尾（用户主动给出具体场景）

**目的：** 验证「用户已经说清楚了 → 不再追问，直接出笔记」

**发送：**
> 我发现跟别人解释一个概念的时候，自己反而理解更深了

**第 1 轮回复时直接给具体场景：**
> 上周给同事讲我们广告投放的归因模型，讲着讲着自己突然想通了一个之前一直没搞清楚的逻辑

**预期行为：**
- [ ] Bot 识别到具体应用场景，提前收尾
- [ ] 不强行凑满 3 轮追问
- [ ] 笔记「个人应用场景」一栏有内容（不是「待补充」）

**记录：**
- Bot 是否识别到这是具体场景？
- 笔记中「个人应用场景」是否正确提取？

---

### 用例 5：/stats 指令

**目的：** 验证 /stats 返回真实统计

**发送：**
> /stats

**预期行为：**
- [ ] Bot 返回统计数据（对话总数、完成率、应用场景率等）
- [ ] 数据与本地 `python3 sync-notes.py --stats` 一致

---

### 用例 6：边界情况

**依次发送以下消息，每条等 Bot 回复后再发下一条：**

| 发送内容 | 预期回复 |
|---------|---------|
| 😊 | 「没看明白你发的，能再说一句吗？」 |
| 先到这里 | 应该走 conversation_end（即使之前没有对话） |

## 三、测评后操作

### 1. 同步笔记到 Obsidian

```bash
cd ~/Desktop/第二大脑Crucible项目
python3 sync-notes.py
```

**检查：**
- [ ] 新笔记出现在 `~/Documents/ObsidianVault/Crucible/` 目录
- [ ] 打开 Obsidian，Crucible 文件夹下能看到新笔记
- [ ] 笔记格式正确（YAML frontmatter + 4 个章节）

### 2. 查看统计

```bash
python3 sync-notes.py --stats
```

**检查：**
- [ ] 对话总数增加了（应该比测评前多 4-6 条）
- [ ] 完成收尾数增加
- [ ] 平均轮数在 2-4 之间（合理范围）
