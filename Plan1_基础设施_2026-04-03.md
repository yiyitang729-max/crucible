# Plan 1: 基础设施 + 飞书消息跑通 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 完成 Crucible 项目的基础设施搭建，让飞书 Bot 能通过 OpenClaw Gateway 收发消息，并验证「Feishu → OpenClaw → Crucible Skill → Feishu」完整链路跑通（Hello World 级别）。

**Architecture:** OpenClaw 作为消息路由层和 Agent 运行时，通过内置 Feishu Channel 以 WebSocket 长连接接收飞书消息，无需公网 URL。Crucible 业务逻辑以 OpenClaw Skill（SKILL.md 指令文件 + 工具命令）的形式接入 Gateway。本 Plan 只建立通道和 Skill 占位文件，不涉及 Socratic 追问、Obsidian 写入等业务逻辑（Plan 2-4 实现）。

**Tech Stack:** Node.js 24+（或 22.16+），OpenClaw CLI（global），飞书开放平台（企业自建应用），Anthropic API（claude-sonnet-4-6）

## Task 2: 安装 OpenClaw 并完成初始化

**Files:**
- `~/.openclaw/openclaw.json`（由 onboard 生成）

- [ ] **Step 1: 全局安装 OpenClaw**

```bash
npm install -g openclaw@latest
```

Expected: 输出末尾包含 `+ openclaw@x.x.x` 或 `added xxx packages`

- [ ] **Step 2: 验证 CLI 可用**

```bash
openclaw --version
```

Expected: 输出版本号，如 `openclaw/0.x.x node/v24.x.x darwin arm64`

如果提示 `command not found`，检查 npm global bin 是否在 PATH：

```bash
npm bin -g          # 输出全局 bin 目录，如 /usr/local/bin 或 ~/.nvm/versions/node/v24.x.x/bin
echo $PATH          # 确认上面的目录在 PATH 中
```

若不在 PATH，添加到 `~/.zshrc`：

```bash
echo 'export PATH="$(npm bin -g):$PATH"' >> ~/.zshrc
source ~/.zshrc
openclaw --version
```

- [ ] **Step 3: 运行 onboard 向导**

```bash
openclaw onboard --install-daemon
```

向导会依次提问，按以下选择：

| 提示 | 选择 |
|------|------|
| AI provider | **Anthropic** |
| API key | 粘贴 `ANTHROPIC_API_KEY`（从 https://console.anthropic.com/settings/api-keys 获取） |
| Default model | `claude-sonnet-4-6`（或选择列表中最新的 Sonnet） |
| Install as daemon (auto-start) | **Yes** |

Expected: 向导结束后输出成功信息，`~/.openclaw/openclaw.json` 文件被创建

- [ ] **Step 4: 验证 Gateway 启动**

```bash
openclaw gateway status
```

Expected: 输出 `running` 或 `Gateway is running on port 18789`

如果未运行，手动启动（前台模式，测试用）：

```bash

```

看到 `Gateway listening` 后，按 `Ctrl+C`，再通过 daemon 方式启动（onboard 已配置）：

```bash
# macOS launchd daemon 方式
openclaw gateway status   # daemon 应该自动启动了
```

- [ ] **Step 5: 确认配置文件**

```bash

```

Expected: 包含 `agent.model` 字段，类似：


## Task 3: 在飞书开放平台创建应用

**Files:**
- 无（浏览器操作）

> 中国大陆飞书用户：https://open.feishu.cn/app  
> 海外 Lark 用户：https://open.larksuite.com/app

- [ ] **Step 1: 创建企业自建应用**

1. 登录飞书开放平台
2. 点击「创建企业自建应用」
3. 填写：
   - **应用名称**：`Crucible`
   - **应用描述**：`AI 思考伙伴 - 个人知识管理助手`
4. 点击「确认创建」，进入应用详情页

- [ ] **Step 2: 复制应用凭证**

1. 左侧菜单 → 「凭证与基础信息」（Credentials & Basic Info）
2. 复制并安全保存以下两项（下一个 Task 要用）：
   - **App ID**（格式：`cli_xxxxxxxxxx`）
   - **App Secret**

⚠️ App Secret 不要提交到 git，不要明文写入项目文件

- [ ] **Step 3: 启用机器人能力**

1. 左侧菜单 → 「应用能力」→「机器人」（Bot）
2. 点击「启用机器人」
3. 确认状态变为「已启用」

- [ ] **Step 4: 配置事件订阅（WebSocket 模式）**

1. 左侧菜单 → 「事件订阅」（Event Subscriptions）
2. 将订阅方式切换为「**使用长连接接收事件**」（WebSocket，无需公网 URL）
3. 点击「添加事件」→ 搜索 `im.message.receive_v1` → 选中 → 确认
4. 点击「保存」

- [ ] **Step 5: 开通权限**

1. 左侧菜单 → 「权限管理」（Permissions）
2. 搜索并申请以下权限：

| 权限名 | 说明 |
|--------|------|
| `im:message:receive_v1` | 接收消息（接收用户发给 Bot 的消息） |
| `im:message:create_v1` | 发送消息（Bot 回复用户） |
| `contact:user.base:readonly` | 读取用户基础信息（识别发消息的是谁） |

3. 点击「批量申请」确认

- [ ] **Step 6: 发布应用**

1. 左侧菜单 → 「版本管理与发布」
2. 点击「创建版本」，填写版本说明 `v0.1 测试版`
3. 点击「申请发布」

> **个人飞书账号说明**：如果没有企业管理员权限，选择「体验版」发布，在「可用范围」里添加自己的飞书账号即可。体验版不需要管理员审批，立即生效。

## Task 4: 配置 OpenClaw Feishu Channel

**Files:**
- `~/.openclaw/openclaw.json`（修改）

- [ ] **Step 1: 备份当前配置**

```bash
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak
cat ~/.openclaw/openclaw.json  # 查看当前内容，确认备份成功
```

- [ ] **Step 2: 添加 Feishu channel 配置**

用编辑器打开 `~/.openclaw/openclaw.json`，将 `channels` 块添加进去：

```json5
{
  agent: {
    model: "anthropic/claude-sonnet-4-6",
  },
  channels: {
    feishu: {
      enabled: true,
      connectionMode: "websocket",  // 长连接，无需公网 URL
      dmPolicy: "pairing",          // 首次私聊需要 pairing 授权
      accounts: {
        main: {
          appId: "cli_xxxxxxxxxx",   // 替换为 Task 3 Step 2 中的 App ID
          appSecret: "xxxxxxxxxx",   // 替换为 Task 3 Step 2 中的 App Secret
          name: "Crucible",
        },
      },
    },
  },
}
```

⚠️ `appId` 和 `appSecret` 填真实值，保留 `json5` 注释格式（OpenClaw 支持 JSON5）

- [ ] **Step 3: 重启 Gateway 使配置生效**

```bash
# 停止当前 Gateway（如果是 daemon）
openclaw gateway stop

# 重新启动
openclaw gateway --port 18789 --verbose
```

查看启动日志，寻找 Feishu 连接相关输出，类似：

```
[feishu] WebSocket connecting to appId: cli_xxxxxxxxxx
[feishu] WebSocket connected
```

如果看到 `connected` 日志，说明凭证正确，Feishu 通道已建立。

如果看到错误（如 `401` 或 `invalid app_id`），回到 Task 3 Step 2 重新确认凭证是否复制正确。

- [ ] **Step 4: 验证 Gateway 正在监听**

新开一个终端窗口（不要关掉 Gateway 的窗口）：

```bash
openclaw gateway status
```

Expected: 输出显示 Gateway running，无错误

## 常见问题排查

**Q: `openclaw command not found`**  
A: npm global bin 目录不在 PATH。见 Task 2 Step 2 的修复方法。

**Q: Gateway 启动时 Feishu 连接报 `invalid credential` 或 `401`**  
A: App ID 或 App Secret 粘贴有误（注意前后空格）。重新从飞书开放平台复制。

**Q: 飞书 Bot 收不到消息**  
A: 检查事件订阅是否配置了 `im.message.receive_v1`，且订阅方式是「长连接」而非 Webhook。

**Q: Bot 不回复**  
A: 检查 Gateway 日志有无错误；检查 `ANTHROPIC_API_KEY` 是否有效（可在 Anthropic Console 查看 API key 状态）。

**Q: Pairing 后 Bot 还是不回复**  
A: 确认 `openclaw.json` 中 `skills: ["crucible"]` 已添加且 Gateway 已重启。
