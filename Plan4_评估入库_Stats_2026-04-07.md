# Plan 4: crucible.db 评估入库 + /stats 统计指令

**Goal:** 每次同步时自动分析对话质量指标，写入本地 SQLite 数据库；飞书 Bot 支持 `/stats` 指令返回真实统计。

**Architecture:** 扩展 sync-notes.py，在同步笔记的同时从飞书消息流中分段对话、计算指标、写入 crucible.db。统计摘要上传到服务器 STATS.md，Bot 的 `/stats` 路由读取该文件返回给用户。

**Tech Stack:** Python 3 + sqlite3（标准库）、飞书 Open API、SSH/SCP（上传 stats）

---

## 实现内容

### 1. 对话分段算法

从飞书消息流中识别独立对话：
- 按时间正序排列所有消息
- 两条消息间隔超过 30 分钟 → 视为不同对话
- 跳过空消息和 HTTP 500 错误消息

### 2. 对话指标计算

每段对话提取以下指标：

| 字段 | 说明 | 计算方式 |
|------|------|---------|
| id | 对话 ID | 第一条消息的 message_id |
| created_at | 开始时间 | 第一条消息的 create_time |
| input_type | 输入类型 | 含 URL → article，否则 → thought |
| turns | 用户消息数 | 统计 sender_type=user 的消息数 |
| completed | 是否完成 | Bot 消息中包含「Crucible 笔记」→ true |
| has_application | 有应用场景 | 笔记中「个人应用场景」不是「待补充」→ true |
| user_msg_chars | 用户总字数 | 所有用户消息字数之和 |
| note_id | 关联笔记 | 包含笔记的 Bot 消息 message_id |

### 3. 数据库结构

`crucible.db` 存放在 Obsidian Vault 根目录（`$OBSIDIAN_VAULT_PATH/crucible.db`）。

```sql
CREATE TABLE conversation_metrics (
    id              TEXT PRIMARY KEY,
    created_at      DATETIME,
    input_type      TEXT,
    turns           INTEGER,
    completed       BOOLEAN,
    has_application BOOLEAN,
    user_msg_chars  INTEGER,
    note_id         TEXT
);

CREATE TABLE note_metrics (
    note_id         TEXT PRIMARY KEY,
    created_at      DATETIME,
    linked_count    INTEGER DEFAULT 0,  -- V2 填入
    searched_hit    INTEGER DEFAULT 0   -- V2 填入
);
```

### 4. /stats 指令

**数据流：**
```
sync-notes.py 计算统计 → 生成 STATS.md → SCP 上传到服务器
用户发 /stats → Bot 读取 STATS.md → 返回内容
```

**统计输出包含：**
- 总览：对话总数、完成率、应用场景率、平均轮数、笔记总数
- 最近 7 天：对话数、完成数、应用场景数
- 最近 30 天：活跃天数

### 5. 使用方式

```bash
# 同步笔记 + 更新 crucible.db + 上传统计
python3 sync-notes.py

# 仅查看统计
python3 sync-notes.py --stats
```

## 验收标准

| 验收项 | 验证方式 |
|--------|---------|
| 对话能被正确分段 | `python3 sync-notes.py` 输出对话数量合理 |
| 指标写入 crucible.db | `sqlite3 crucible.db "SELECT * FROM conversation_metrics"` |
| --stats 输出统计 | `python3 sync-notes.py --stats` |
| Bot /stats 返回数据 | 飞书发 `/stats` 收到统计回复 |
| 统计数据准确 | 对比本地 --stats 与飞书 /stats 输出一致 |
